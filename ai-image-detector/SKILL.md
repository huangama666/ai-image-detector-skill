# AI Image Detector

Use this skill to detect whether an image is AI-generated or AI-composited. Keep hard evidence and visual/content suspicion separate.

## Quick Start

Run the bundled inspector:

```bash
PYTHONPATH=/path/to/ai-image-detector/scripts python3 -m ai_image_detector.cli /path/to/image.png --ocr --content-assessment likely_ai --content-note "画面符合生成式视觉特征" --output-dir /tmp/ai-image-report --artifacts-dir /tmp/ai-image-report
```

For multiple files:

```bash
PYTHONPATH=/path/to/ai-image-detector/scripts python3 -m ai_image_detector.cli image1.png image2.jpg --output-dir /tmp/ai-image-report
```

The command prints JSON and, when `--output-dir` is set, writes:

- `*-ai-image-detection-report.json`
- `*-ai-image-detection-report.html`
- optional `*-fft-spectrum.png` when `--artifacts-dir` is set

## Workflow

1. Preserve the original path and do not overwrite the source image.
2. Run the inspector first without OCR if speed matters; add `--ocr` when visible logos/text/watermarks may matter.
3. Read `verdict.label`, `ai_indicators`, `metadata`, `embedded_signals`, and `content_assessment`.
4. Summarize hard-evidence findings as:
   - `strong_ai_evidence`: generator metadata, C2PA/Content Credentials markers, or clear embedded AI workflow fields.
   - `possible_ai_evidence`: OCR-visible marks or weaker heuristic strings.
   - `no_ai_evidence`: no supported evidence found.
5. Add visual/content suspicion separately when the image looks AI-generated but hard metadata is missing, especially for screenshots, social-platform exports, or recompressed images.
6. State limitations clearly, especially for screenshots, WeChat/social compression, format conversion, and private invisible watermarks.

## Evidence Rules

Treat these as strong evidence:

- PNG text fields such as `parameters`, `prompt`, `workflow`, or `Comment` containing Stable Diffusion, ComfyUI, NovelAI, Midjourney, DALL-E/OpenAI, Firefly, or similar markers.
- TC260 AIGC XMP metadata such as `TC260:AIGC`, `tc260.org.cn/ns/AIGC`, or `Label: 1`.
- C2PA, JUMBF, or Content Credentials markers in the file bytes.
- XMP/JPEG comments containing known AI generator metadata.

Treat these as weak evidence:

- OCR text that says "generated by AI", "OpenAI", "Midjourney", "Firefly", or similar visible marks.
- Binary strings that look like prompts but are not attached to clear metadata fields.
- Frequency spectrum or compression artifacts by themselves.

Do not claim:

- That an image is definitely AI-generated solely from visual artifacts.
- That private invisible watermarks were extracted unless an official detector or embedded marker confirms it.
- That absence of evidence proves the image is human-made.

## Output Summary Pattern

When reporting to the user, include:

- Hard-evidence verdict label and score.
- Visual/content suspicion label and reasons, if provided.
- Strongest 3-5 indicators with source fields.
- Whether C2PA/Content Credentials were present.
- Whether OCR was run and what it found.
- Where JSON/HTML reports were saved, if generated.
- A short limitation note.

Example:

```text
硬证据：strong_ai_evidence，score 75。
视觉/内容判断：高度疑似 AI，理由：画面为生成式 3D IP 角色，光影和材质高度统一。
证据：PNG parameters 字段包含 Stable Diffusion 生成参数，包括 Negative prompt、Steps、Sampler。
C2PA：未发现。
OCR：未运行。
报告：/tmp/ai-image-report/example-ai-image-detection-report.html
限制：如果图片经过截图或平台压缩，元数据和隐形水印可能已经丢失。
```
